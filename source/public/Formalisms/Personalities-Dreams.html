    Personalities Dreams

    Mar 17, 20252 min read

    • LLM
    • cog-dev

    A personalities dreams should be performed through a series of function.

    First we recognize some Φ composed of some set LLM calls on a set of tuples {ϕ(P1​,r1​),..,ϕ(Pi​,rN​)}. Where the list contains n distinct P and N distinct r. The order of operations is not necessarily direct.

    Φ=ϕ(P1​,r1​)→ϕ(P2​,r2​)→ϕ(P1​,r3​)→⋯→ϕ(Pn​,rj​)→ϕ(Pi​,rN​)→ofinal​ ΦPi​​={ϕ(Pi​,rj​) for any valid j=1,..,N} ΦPi​​ is essentially the space that a given sub-personality in a personality-matrix covers. This may be represented as some other set of logical/semantic meanings such as: ΦPi​​={ϕ(Pi​,rj​)⟺j=valid}↦oi​

    \mathcal{V}_{correct}(\mathcal{o}_{i})=\begin{dcases*} \text{correct}, & \text{if $\exists \hspace{0.1 cm} I_{\Phi} \in \mathcal{o}_{i} $}\\ \text{incorrect}, & \text{if $ \exists \hspace{0.1 cm} I_{\Phi} \not \in \mathcal{o}_{i}$} \end{dcases*}$$

    \mathcal{V}{valid}(\mathbb{o}{i})=\begin{dcases*} \text{valid}, & \text{if ∃IΦ​∈oi​}\ \text{invalid}, & \text{if ∃IΦ​∈oi​} \end{dcases*}$$ and this can be used to build a matrix of Boolean type values

    For all personalities in a given Φ there are distinct bounds that are considered the “norm”. These are calculated via the application of Thermodynamics of LLM conformational space and the usage of Detecting information in personality matrices

    To dream requires Language model communication

    The process of dreaming would thus be:

    Select your dream topic:

    1. Take the tuples of (Pi​,rj​) from Pi,j​ for every i and j. Create your dream:
    2. Run ϕ(Pi​,rj​) at high temperature (↑T) to make oi,j,dream​ Narrativize your dream:
    3. Run ϕnarrative​(Pi​,oi,j,dream​) ask to “make a narrative from oi,j,dream​” to make oi,j,narrative​ Dream interpretation:
    4. Run ϕmeaning​(Pi​,oi,j,narrative​) and ask “what does oi,j,narrative​ mean” to make oi,j,meaning​ Lucid Dreaming:
    5. Run ϕ(Pi​,oi,j,narrative​,oi,j,meaning​) and ask “what would you change in oi,j,narrative​ given oi,j,meaning​ ” → “write that story” to get oi,j,lucid​
    6. Repeat step 3-5 to get oi,j,refined​ Compiling
    7. Repeat steps 1-6 for all i,j and collate those stories into one final overarching story ϕ(Pi​,oi,j,refined​)↦oi,j,final​ Recording:
    8. Write oi,final​ into IS​ to update the self
